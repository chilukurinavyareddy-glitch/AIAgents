# Skill: SQL Server

Use when work touches SQL Server schema, scripts, migrations, seed data, or query behavior.

## Guidance

- Do not create schema changes without explicit requirement.
- Prefer reviewed migration scripts over ad hoc database edits.
- Keep seed data separate from schema scripts.
- Consider indexes and constraints for data integrity and performance.
- Never run production database changes automatically.

## Outputs

- Migration scripts.
- Seed scripts.
- Data access validation notes.
