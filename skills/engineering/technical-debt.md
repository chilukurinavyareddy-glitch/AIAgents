# Skill: Technical Debt Management

Use when feature work exposes cleanup, shortcuts, duplication, or deferred risk.

## Guidance

- Separate must-fix blockers from acceptable follow-up debt.
- Do not hide technical debt in successful execution summaries.
- Avoid broad refactors during narrow feature work.
- Create follow-up notes when debt is real but not blocking.

## Outputs

- Technical debt notes.
- Follow-up recommendations.
- Blocker decision when debt is unsafe.
