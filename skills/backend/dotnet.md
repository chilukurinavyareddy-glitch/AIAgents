# Skill: .NET Backend

Use when implementing .NET backend features.

## Guidance

- Follow the selected project's target framework and solution structure.
- Keep API, services, repositories, domain, and common concerns separated according to project standards.
- Use dependency injection for services.
- Keep controllers thin.
- Prefer typed request and response models.
- Do not hardcode environment-specific values.

## Outputs

- Services.
- Controllers or endpoints.
- Models/contracts.
- DI registration when needed.
- Tests.
