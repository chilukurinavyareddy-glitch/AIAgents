# Skill: Backend Error Handling

Use when implementing APIs, services, validation, or integration boundaries.

## Guidance

- Return clear, typed error responses.
- Do not leak internal exception details.
- Distinguish validation, authorization, not found, conflict, and server errors.
- Log server-side exception context without exposing secrets or sensitive data.
- Keep error handling consistent with the selected project's standards.

## Outputs

- Error response contract.
- Validation and negative test cases.
- Logging and observability notes.
