# Skill: API Design

Use when defining or implementing API contracts.

## Guidance

- Keep routes clear and resource-oriented.
- Return typed responses.
- Validate inputs before business logic.
- Use appropriate status codes.
- Avoid leaking internal exception details.
- Keep backward compatibility in mind for existing routes.

## Outputs

- API route contract.
- Request and response models.
- Error handling notes.
- API tests.
