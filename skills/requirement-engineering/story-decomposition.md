# Skill: Story Decomposition

Use when breaking feature intent into implementation-ready work.

## Guidance

- Keep stories focused on user-visible or system-deliverable outcomes.
- Avoid unnecessary Jira hierarchy.
- Reuse existing open Jira work when intent matches.
- Split frontend, backend, database, and QA work only when it improves execution clarity.

## Outputs

- Requirement summary.
- Acceptance criteria.
- Story or task breakdown.
