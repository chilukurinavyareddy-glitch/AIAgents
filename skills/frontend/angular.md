# Skill: Angular Frontend

Use when building Angular UI features.

## Guidance

- Follow the selected project's Angular version and project structure.
- Prefer feature folders for route-level features.
- Keep route configuration explicit and minimal.
- Use typed services for API calls.
- Keep components focused on view state and interaction.
- Use standalone or NgModule patterns according to the existing application.
- Do not introduce a new frontend framework.

## Outputs

- Route or navigation changes when required.
- Component, template, styles, models, and services.
- Unit tests.
