# Skill: UX Review

Use when reviewing or building user-facing workflows.

## Guidance

- Keep primary workflows obvious and efficient.
- Show loading, empty, success, and error states.
- Avoid unnecessary explanation text inside the application.
- Keep visual hierarchy clear.
- Make repeated operational workflows easy to scan.
- Match UI density to the product context.

## Outputs

- UX assumptions.
- UI state checklist.
- Review findings.
