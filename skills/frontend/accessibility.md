# Skill: Frontend Accessibility

Use when implementing or reviewing user interface work.

## Guidance

- Use semantic HTML where possible.
- Ensure form controls have labels.
- Support keyboard navigation for interactive controls.
- Keep focus states visible.
- Use accessible names for buttons, links, and icons.
- Check color contrast for important text and controls.

## Outputs

- Accessibility notes.
- UI review findings.
- Tests or manual checks when applicable.
