# Skill: Performance Testing

Use when a feature affects data volume, rendering performance, API latency, or repeated operational workflows.

## Guidance

- Identify expected data size and request frequency.
- Check obvious N+1 queries, repeated API calls, excessive rendering, and expensive filters.
- Add lightweight performance checks when practical.
- Use performance budgets when project standards define them.
- Escalate when realistic load testing is required.

## Outputs

- Performance risk notes.
- Performance test or benchmark plan.
- Review findings.
