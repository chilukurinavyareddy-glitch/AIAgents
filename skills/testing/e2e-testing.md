# Skill: End-to-End Testing

Use when user workflows span UI, API, persistence, and routing.

## Guidance

- Cover the highest-value user journeys.
- Avoid brittle selectors.
- Keep E2E tests focused and small.
- Prefer stable test data setup and teardown.
- Run E2E tests only when environment readiness is confirmed.

## Outputs

- E2E scenario list.
- E2E test files or deferred E2E rationale.
- Environment readiness notes.
