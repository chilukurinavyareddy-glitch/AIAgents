# Skill: Integration Testing

Use when work crosses APIs, services, repositories, databases, or frontend/backend boundaries.

## Guidance

- Test important boundaries, not every implementation detail.
- Mock external services unless the scenario explicitly requires live integration.
- Cover success, validation failure, authorization failure, and not-found/conflict paths.
- Keep integration tests deterministic and isolated.
- Record skipped integration risks explicitly.

## Outputs

- Integration test plan.
- Integration test files.
- Boundary validation evidence.
