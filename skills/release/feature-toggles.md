# Skill: Feature Toggles

Use when a feature should be releasable separately from being visible or fully enabled.

## Guidance

- Use feature toggles for risky, incremental, or staged functionality when project standards support them.
- Keep toggle naming clear and scoped.
- Avoid long-lived toggles without cleanup ownership.
- Test enabled and disabled behavior when a toggle is used.

## Outputs

- Toggle decision.
- Enabled/disabled test plan.
- Cleanup note.
