# Skill: Rollback Strategy

Use before merge, release validation, or database-affecting changes.

## Guidance

- Identify how to revert application code safely.
- Identify migration rollback constraints.
- Keep deployment and rollback separate from local publish artifacts.
- Do not merge changes with unknown rollback risk when they affect auth, database, payment, or critical workflows.

## Outputs

- Rollback notes.
- Release risk.
- Merge readiness input.
