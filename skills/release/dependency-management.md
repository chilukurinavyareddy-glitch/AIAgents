# Skill: Dependency Management

Use when package, framework, SDK, or library changes are proposed.

## Guidance

- Avoid unrelated dependency upgrades during feature work.
- Check compatibility with the selected project stack.
- Treat security updates separately when they are urgent.
- Record breaking changes and rollback notes.
- Run full build and tests after dependency changes.

## Outputs

- Dependency change summary.
- Compatibility notes.
- Validation evidence.
