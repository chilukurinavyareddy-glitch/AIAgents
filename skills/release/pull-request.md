# Skill: Pull Request Automation

Use when creating, reviewing, or updating pull requests.

## Guidance

- Target the configured integration branch, normally `development`.
- Add configured default reviewers using stable identifiers.
- Include validation evidence in the PR.
- Reuse existing open PRs from the same branch.
- Do not merge before review, tests, score, confidence, and security gates pass.

## Outputs

- PR URL.
- Reviewer result.
- Merge readiness notes.
