# Skill: DevSecOps

Use when feature work, review, release, or automation touches security-sensitive behavior.

## Guidance

- Shift security review left into requirements, architecture, implementation, and QA.
- Identify authentication, authorization, data exposure, injection, secrets, dependency, and logging risks early.
- Treat security blockers as merge blockers.
- Require tests or review evidence for security-sensitive changes.
- Never bypass security gates for automation convenience.

## Outputs

- Security risk notes.
- Security test expectations.
- Merge blocker decision when needed.
