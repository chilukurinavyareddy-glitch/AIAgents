# Skill: Scalability

Use when designing features expected to grow in users, data volume, integrations, or request rate.

## Guidance

- Identify expected scale assumptions.
- Keep module boundaries clear.
- Avoid designs that require unrelated modules to change together.
- Consider pagination, filtering, indexing, caching, and async processing when relevant.
- Do not overengineer beyond current project maturity.

## Outputs

- Scalability assumptions.
- Architecture tradeoffs.
- Future scaling notes.
