# Skill: Resiliency

Use when work touches external services, retries, failure handling, background work, or critical workflows.

## Guidance

- Define expected failure modes.
- Avoid silent failures.
- Use retries only when safe and bounded.
- Keep user-facing error states clear.
- Record recovery and fallback behavior.
- Do not hide data loss or partial completion.

## Outputs

- Failure-mode notes.
- Recovery behavior.
- Test cases for failure paths.
