# Skill: Architecture Review

Use when reviewing architectural fit, boundaries, contracts, shared resources, or cross-module risk.

## Guidance

- Verify the implementation follows the selected project's architecture.
- Check that shared routing, startup, dependency injection, navigation, and configuration changes are intentional.
- Confirm module locks and shared-resource locks are appropriate for parallel work.
- Identify coupling, duplication, and contract drift.
- Escalate security-sensitive architecture changes.

## Outputs

- Architecture findings.
- Shared-resource risk notes.
- Review decision input.
