# Skill: Performance Review

Use when reviewing API, UI, database, or repeated workflow changes.

## Guidance

- Check avoidable repeated calls, expensive client filtering, unbounded queries, large payloads, and missing pagination.
- Check UI rendering and change detection risks for large lists.
- Identify database indexing or query risks.
- Avoid premature optimization when risk is low.

## Outputs

- Performance findings.
- Risk level.
- Follow-up recommendations.
