# Skill: Code Review

Use during implementation review before git handoff.

## Guidance

- Prioritize correctness, security, maintainability, and regression risk.
- Verify tests match the changed behavior.
- Check project context and engineering standards.
- Avoid saving one-time personal preferences as reusable learnings.
- Identify reusable engineering learnings only when they help future projects.

## Outputs

- Findings.
- Security notes.
- Test coverage notes.
- Approval decision.
